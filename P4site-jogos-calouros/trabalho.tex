\subsection{Desenvolvimento do Jogo Duelo de Reflexo}

O jogo Duelo de Reflexo foi desenvolvido utilizando HTML, CSS e JavaScript
e permite a participação simultânea de dois jogadores. O objetivo consiste em
pressionar a tecla correspondente ao jogador o mais rapidamente possível após
a exibição do sinal de início da rodada.

Antes de cada rodada, o sistema aguarda um intervalo de tempo aleatório. Caso
um jogador pressione sua tecla antes do sinal, a jogada é considerada inválida
e o participante é penalizado por ter quebrado as regras. Após o início válido,
o programa registra o tempo de reação e determina o dano causado ao adversário.

A estrutura HTML foi utilizada para organizar os elementos da interface,
incluindo o cenário da luta, os personagens, as barras de vida, as mensagens
de orientação e os controles da partida.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.75\textwidth]{imgs/HtmlReflexo.png}
    \caption{Estrutura HTML utilizada no jogo Duelo de Reflexo.}
    \label{fig:html-reflexo}
\end{figure}

A estilização visual foi implementada utilizando CSS, responsável pelo
posicionamento dos personagens, organização das barras de vida, composição
do cenário e aplicação dos efeitos visuais apresentados durante os ataques.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.75\textwidth]{imgs/ReflexoCss1.png}
    \caption{Trecho da estilização CSS utilizada no jogo Duelo de Reflexo.}
    \label{fig:css-reflexo}
\end{figure}

A lógica da aplicação foi implementada em JavaScript. O programa controla
o intervalo aleatório entre as rodadas, identifica o jogador que pressionou
a tecla primeiro, calcula o tempo de reação e determina o dano causado.

O dano aplicado varia de acordo com a velocidade da resposta. Tempos menores
resultam em ataques mais fortes, enquanto respostas mais lentas causam uma
quantidade menor de dano. O código também atualiza as barras de vida,
controla o encerramento da partida e exibe o jogador vencedor.

\subsection{Desenvolvimento do Jogo da Memória}

O jogo da memória foi desenvolvido utilizando HTML, CSS e JavaScript. O
objetivo do jogador é encontrar todos os pares de cartas com imagens
correspondentes. Para isso, duas cartas podem ser reveladas a cada jogada.

A estrutura HTML foi utilizada para criar o tabuleiro e organizar
individualmente as cartas. Cada carta possui uma face frontal, contendo uma
imagem, e uma face traseira, exibida enquanto a carta ainda não foi selecionada.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.75\textwidth]{imgs/memoriaHtml.png}
    \caption{Estrutura HTML utilizada na construção do jogo da memória.}
    \label{fig:html-memoria}
\end{figure}

A estilização foi realizada utilizando CSS. Foram definidos o posicionamento
das cartas, as dimensões do tabuleiro, a aparência das faces e o efeito de
rotação utilizado para representar visualmente a abertura e o fechamento de
cada carta.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.70\textwidth]{imgs/memoriaCss.png}
    \caption{Trecho da estilização CSS utilizada no jogo da memória.}
    \label{fig:css-memoria}
\end{figure}

A lógica da partida foi implementada em JavaScript por meio da manipulação do
DOM e do tratamento de eventos de clique. Quando a primeira carta é
selecionada, ela é armazenada temporariamente. Após a seleção da segunda
carta, o programa compara os identificadores associados às duas imagens.

Quando as cartas formam um par, seus eventos de clique são removidos e elas
permanecem abertas. Caso sejam diferentes, o tabuleiro é temporariamente
bloqueado e as cartas são fechadas novamente após um pequeno intervalo de
tempo. Esse bloqueio impede que outras cartas sejam selecionadas durante a
animação.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.75\textwidth]{imgs/memoriaJs.png}
    \caption{Trecho da implementação em JavaScript do jogo da memória.}
    \label{fig:js-memoria}
\end{figure}

As cartas são embaralhadas automaticamente no início da partida, alterando
sua posição no tabuleiro. A Figura \ref{fig:jogo-memoria} apresenta o resultado
visual do jogo em execução.

\begin{figure}[h]
    \centering
    \includegraphics[width=0.80\textwidth]{imgs/jogoMemoria.png}

\section{Resultados Obtidos}

Ao término do projeto foi desenvolvida uma plataforma denominada \textit{Play UEL}, composta por cinco jogos digitais implementados integralmente utilizando HTML, CSS e JavaScript. Todos os jogos foram integrados em uma interface única, permitindo ao usuário navegar facilmente entre as diferentes atividades propostas.

Cada jogo apresenta características próprias de interação, exigindo a implementação de diferentes estruturas de dados, manipulação de eventos, atualização dinâmica da interface e controle do estado da aplicação. Durante o desenvolvimento foram aplicados conceitos relacionados à programação web, manipulação do DOM, estilização responsiva, lógica de programação e organização modular do código.

Os testes realizados demonstraram o correto funcionamento das funcionalidades implementadas, incluindo detecção de vitórias, derrotas e empates, atualização automática das interfaces, reinicialização das partidas e interação entre jogadores. O projeto atingiu os objetivos propostos, resultando em uma plataforma funcional, visualmente padronizada e de fácil utilização.
